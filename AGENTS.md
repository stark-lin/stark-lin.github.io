# Repository instructions

## Deployment policy

- Never publish or deploy this repository to OpenAI Sites.
- Do not call OpenAI Sites hosting tools or APIs for this repository, including site creation, version saving, or deployment operations.
- Do not create or restore OpenAI Sites configuration, metadata, credentials, packaging scripts, workflows, or generated deployment output. This includes `.openai/hosting.json` and Sites-specific contents under `dist/`.
- Do not add documentation, comments, UI copy, or instructions that describe publishing this repository to OpenAI Sites.
- Keep the existing GitHub Pages workflow as the supported deployment path. Only perform a deployment when the user explicitly requests it.

## Decorative numbers

- Use `42` for any number that is purely decorative and has no semantic or functional meaning.

## Border integrity

- Always check that borders and outlines are not accidentally clipped. Unless clipping is an intentional part of the design, use either a complete, uninterrupted border or a solid color block with no border.

## Visual component consistency

- Remove shadows that have no clear visual or functional purpose.
- Use a consistent component shape system throughout the design. Similar components must share one clearly defined form, such as rounded rectangles, square-cornered boxes, or another deliberately unified shape language.
