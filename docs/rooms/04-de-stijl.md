# Room 04：风格派 / De Stijl

## 1. 展厅身份

| 字段 | 值 |
| --- | --- |
| Room Number | `04` |
| Style ID | `de-stijl` |
| 中文名 | 风格派 |
| English Name | De Stijl |
| 历史时期 | 1917–1931 |
| 一句话定义 | 一个由水平线、垂直线、非对称矩形和可见网格构成的持续数字建筑。 |

本展厅把同一份个人档案分配到一个贯穿整页的正交建筑中。它的核心不是机械套用三原色，而是水平与垂直的绝对关系、可见的不等网格、真实保留的空白矩形和非对称平衡。

本文件只定义 Room 04 的结构与表达；内容、配色和随机流仍由全局系统提供，并遵守[通用设计规范](../GENERATIVE_PORTFOLIO_DESIGN_SPEC.md)。

## 2. 核心原则

### 2.1 必须特征

* 贯穿整页、可见且真正分配内容空间的正交网格；
* 不等宽列、不等高矩形和非对称重量关系；
* 主结构线、section 线和内容辅助线三种清楚等级；
* 每个 section 至少包含一个空白平面和一个强调平面；
* 所有结构运动只沿水平或垂直方向；
* 去色和关闭动画后，仍能通过网格建筑识别。

### 2.2 禁止特征

* 把三原色、黑线和白底当成唯一风格条件；
* 斜线、圆形、旋转文字、斜向移动或圆形扩散；
* 平均四宫格、均匀 12 列成品布局或标准 Dashboard；
* 圆角卡片、胶囊标签、阴影悬浮和弹性动画；
* 自动填满所有空白单元；
* 用随机面积暗示项目的重要程度。

## 3. 与易混淆展厅的区别

### 3.1 风格派 ≠ 国际风格

风格派让正交网格直接成为可见构图，强调不等矩形和非对称重量；国际风格更侧重中性、模块化和清晰信息秩序。Room 04 必须保留显著的不均衡平面，不能变成整齐企业网格。

### 3.2 风格派 ≠ 纯粹主义

风格派的线条真正切割并分配连续页面空间；纯粹主义的轴线可以隐含，重点是少量对象的稳定比例。Room 04 不能只保留几块精致色块而失去贯穿网格。

### 3.3 风格派 ≠ 构成主义

风格派只使用水平、垂直和矩形重分配；构成主义使用对角轴、切角和推进方向。Room 04 禁止一切斜向结构与旋转文字。

## 4. 整页结构

页面遵守“首尾固定、中间可变”：

```text
Hero（固定）
↓
Projects / Experience / Education / Stack / Exhibition Note
（由 seed 决定顺序）
↓
Contact（固定）
↓
Roll Again（固定且位于最后）
```

桌面端以不等列建立贯穿网格，可使用以下比例作为基础变体：

```css
grid-template-columns:
  minmax(24px, 0.5fr)
  1fr
  1.5fr
  0.75fr
  2fr
  minmax(24px, 0.5fr);
```

主垂直线必须跨越多个相邻 section 延续；section 可以重新分配单元，但不能每段完全更换网格。线条分三级：主结构线较粗、section 线中等、内容辅助线较细。

中段 section 从以下模板中选择：

* 左窄右宽；
* 上下切割；
* 交错矩形；
* 边缘索引。

顺序、模板和格子占比由 Room 04 的独立 seeded 随机流确定，DOM 顺序与视觉顺序必须一致。空白格是必需结构，不得被内容自动填充。

## 5. Hero

Hero 高度为 `100svh`，由不等矩形切割。必须包含：

* 姓名所在的最大文字平面；
* 相邻矩形中的全局主介绍与可选次级介绍；
* 窄条中的 Room Number；
* 至少一个纯空白矩形；
* 一个承担主要重量的大强调平面；
* 一个用于平衡的小强调平面。

线条必须真正形成 Hero 的内容区域，而非覆盖在背景上的装饰。允许的 seeded 变体为：最大矩形左/右、主垂直线位置、空白平面上/下、姓名横向/纵向视觉分布、强调平面占比。

禁止斜线、圆形、旋转文字和平均四宫格。纵向视觉分布必须通过网格换行实现，不能逐字旋转或改变屏幕阅读器文本。

## 6. Navigation

导航直接嵌入页面网格，每个入口占据不同宽度矩形：

```text
STARK LIN | ENTRY | ARCHIVE | CONTACT | AGAIN | EN / 中文
```

当前阶段使用填充平面或加粗边界，并同步 `aria-current`。滚动后导航保持在顶部并继续共享主网格线。

移动端导航可占两行不等网格，不使用覆盖式菜单；入口顺序与桌面一致，文字不能只用图标替代。

## 7. Personal Archive 与中段 section

每个中段 section 包含编号、名称、正文、一个空白平面和一个强调平面。主要边界线延续，内容按本次模板重新分配。

### 7.1 Projects

每个项目被分配进一组相邻的不等矩形，包含编号、标题、类型、真实描述、技术和链接；字段不必处于同一容器，但必须共享边界与语义关联。

项目尺寸只能由描述长度、字段数量和预设构图顺序决定，不得暗示质量排名或重要程度。

hover/focus 可交换相邻矩形的填充、加粗某条边界或显现描述。点击展开时，一个或多个网格单元沿水平或垂直方向扩张，相邻单元重新分配；不得覆盖整页或打开模态框。

### 7.2 Experience

经历使用连续横向条带，按不等列呈现年份、角色、组织和描述。条带跨度可以变化，但字段顺序和日期事实必须清楚。

### 7.3 Education

教育背景可以占据一个完整大矩形作为结构锚点，完整显示学校、专业、学位与日期。

### 7.4 Working Stack

技术栈使用正交矩阵。技术项可占不同宽度，但不得使用标签胶囊，也不得让面积代表熟练度。

### 7.5 Room Introduction / Exhibition Note

展厅说明是中段的独立 section，由 seed 决定位置。名称、历史时期、策展说明、当前网格变体和 seed 可拆到相邻格子，并至少保留一个空白格；拆分后的 DOM 和屏幕阅读器顺序仍需构成完整说明。

## 8. Contact

Contact 是固定末段的第一部分，以大尺度网格切割呈现 Email、GitHub、LinkedIn 与必要职业链接。每个链接可以占据完整矩形并扩大点击区域，但可见文字必须是实际目标，不使用社交图标。

Contact 与 Roll Again 必须是两个语义独立的 section。视觉上可以共享边界，但不得把 Roll Again 混入 Contact 的 DOM 或让屏幕阅读器误认为联系方式。

## 9. Roll Again

Roll Again 固定在 Contact 后并作为页面最后一个主要 section。它是网格中最大或第二大的交互平面，包含当前 seed、主动作标签和明确的“生成另一个版本”说明。

辅助文案可从 `RECOMPOSE THE GRID`、`ANOTHER PLAN`、`NEW RECTANGULAR ORDER`、`ROLL ANOTHER ROOM` 中选择。

hover/focus 时周围网格线可以重分配，空白格和按钮格可以交换面积，但文字基线与可点击区域必须稳定。激活后网格线收缩到主轴，新网格沿水平或垂直方向展开；Reduced Motion 下直接切换最终布局。

## 10. 色彩关系

颜色只能通过全局令牌取得。Room 04 规定：

* 一个主背景平面；
* 一个主要强调色；
* 一个次强调色；
* 一个小面积平衡色；
* 线条使用独立结构色；
* 同屏至少一个未填充或与背景同色的矩形；
* 颜色不均匀分布，大色块与小色块形成明确重量差；
* 不写死三原色，不使用渐变或阴影。

## 11. 动效、交互与可访问性

动效关键词为切割、延伸、分配、收缩、扩张和重新规划。只允许水平移动、垂直移动、宽高变化、填充切换和边界生长。

* 所有可点击平面使用真实 link 或 `button`；
* 项目展开控件同步 `aria-expanded`；
* 视觉拆分到多个格子的标题或说明保留连续语义 DOM；
* 装饰性空格、重复文字和线段使用 `aria-hidden="true"`；
* focus 同时使用边界加粗、内描边或清楚文字标记，不能只改填充色；
* 网格重分配不得改变键盘焦点位置或 DOM 阅读顺序；
* 线条与强调平面不得遮挡链接。

### 11.1 Reduced Motion

Reduced Motion 下禁用网格线生长、单元交换和收缩展开转场；展开内容直接占据最终网格，填充状态可即时切换。静态网格、空白平面与层级必须完整保留。

## 12. 响应式规则

移动端保留三列或四列不等宽网格，例如“窄边栏 / 主内容 / 次内容 / 窄边栏”，不能完全退化成单列卡片流。

* 主垂直线至少跨越两个相邻 section；
* 导航可占两行网格；
* 长正文允许跨越主内容与次内容列；
* Contact 链接保持足够触摸面积；
* 空白矩形数量可减少，但每个主要 section 至少保留一个；
* 边距为 `16–24px`，不产生意外横向滚动；
* 所有运动仍严格沿水平或垂直方向。

中英文可使用不同单元跨度、字号与换行，但 section 顺序、核心事实、主网格关系和交互职责必须一致。

## 13. 确定性随机与上限

| 参数 | 安全集合或上限 |
| --- | --- |
| 桌面基础网格 | 3 套经过验证的不等 6 列比例 |
| 移动网格 | 3 列或 4 列不等比例 |
| section 模板 | 4 种 |
| 单个 section 强调平面 | 1–2 个 |
| 单个 section 必需空白平面 | 至少 1 个 |
| 线条等级 | 固定 3 级 |
| 允许运动方向 | 水平或垂直 |
| 旋转角度 | 仅 `0deg` |
| 持续动画 | `0` 个 |
| 项目展开 | `180–500ms` |

主垂直线只能在相邻预设位置间变化，同一页面不得每个 section 完全改线。所有选择必须来自 `deriveRng(seed, "room-04-layout")`、`room-04-decoration` 和 `room-04-motion` 等隔离随机流，禁止直接使用 `Math.random()`。

## 14. 验收标准

除[通用验收标准](../spec/26-acceptance-criteria.md)外，Room 04 还必须通过：

### 14.1 去色测试

* 去色后，不等网格、线条等级、空白矩形和非对称平衡仍清楚；
* 不依赖三原色即可识别；
* 链接、当前导航和 focus 仍通过边界与文字可辨。

### 14.2 静态测试

* 关闭全部动画后，网格仍完整分配内容；
* 项目展开、导航和 Roll Again 无动画也可使用；
* 不存在依赖线条生长后才出现的核心内容。

### 14.3 风格与结构测试

* 网格线真正决定内容区域，空白矩形承担平衡作用；
* 页面不像 Dashboard、平均卡片墙或仅换三原色的模板；
* 移动端仍保留不等矩形与主线连续性；
* Hero 固定在开头，Exhibition Note 只在中段变化，Contact → Roll Again 固定在结尾。
