# Room 03：至上主义 / Suprematism

## 1. 展厅身份

| 字段 | 值 |
| --- | --- |
| Room Number | `03` |
| Style ID | `suprematism` |
| 中文名 | 至上主义 |
| English Name | Suprematism |
| 历史时期 | 1910s–1920s |
| 一句话定义 | 一个由漂浮几何、失重空间和开放距离组成的非对象化网页场域。 |

本展厅让同一份个人档案摆脱传统网页栏目和容器，通过空白、尺度、方向和视觉重量建立阅读场域。它不是随机撒形状，也不是动态屏保；所有空间位置都来自预先设计并经过可用性验证的构图。

本文件只定义 Room 03 的结构与表达；内容、配色和随机流仍由全局系统提供，并遵守[通用设计规范](../GENERATIVE_PORTFOLIO_DESIGN_SPEC.md)。

## 2. 核心原则

### 2.1 必须特征

* 大面积具有结构意义的开放空间；
* 文字与几何体通过距离、尺度和方向形成视觉重量关系；
* 隐形空间网格与有限构图预设，而非逐元素任意定位；
* 内容像空间对象，不依赖统一容器边框；
* 核心正文稳定，只有少量形体轻度漂移或让位；
* 静态和去色状态下仍能识别空间张力。

### 2.2 禁止特征

* 对每个元素随机生成坐标、角度、尺度或 z-index；
* 把自由空间误解为内容散乱、不可查找或 DOM 顺序混乱；
* 用持续粒子、屏保动画、强视差或正文漂浮制造“失重”；
* 依赖卡片、容器边框、阴影或透明叠色组织所有信息；
* 用几何体遮挡正文、链接或 focus 状态；
* 让空白导致无法在合理滚动长度内找到 Contact 和 Roll Again。

## 3. 与易混淆展厅的区别

### 3.1 至上主义 ≠ 构成主义

至上主义通过离心关系、开放距离和悬浮形体建立非对象空间；构成主义通过对角轴、命令带和机械模块推动行动。Room 03 不使用连续命令口号、厚重结构板或统一推进轴。

### 3.2 至上主义 ≠ 纯粹主义

至上主义允许受控倾斜、元素分离和失重感；纯粹主义依赖稳定双轴、低密度比例与禁止旋转的静物关系。Room 03 的空白是场域，Room 01 的空白是比例平面。

### 3.3 至上主义 ≠ 随机生成艺术

本展厅不会无限生成图形，也不让 seed 随意扰动每个坐标。seed 只能选择完整构图预设及少量安全偏移；个人内容始终可读、可复制、可定位。

## 4. 整页结构

页面遵守“首尾固定、中间可变”：

```text
Hero（固定）
↓
Projects / Experience / Education / Stack / Exhibition Note
（由 seed 决定顺序）
↓
Contact（固定）
↓
Roll Again（固定且位于最后）
```

桌面端使用 12 列、8 个纵向位置带组成的隐形空间网格。网格不直接显示，但所有文字和形体必须落在预设位置上。每个中段 section 高度只能从 `90svh`、`120svh`、`150svh` 中选择，一个视口只呈现少量核心内容。

构图从以下八个完整预设中选择：

1. 左重右空；
2. 上重下空；
3. 对角漂移；
4. 中央空洞；
5. 四角张力；
6. 水平散布；
7. 垂直上升；
8. 离心构图。

seed 只选择预设、镜像状态和受限偏移，不逐元素随机定位。视觉顺序必须与 seeded DOM 顺序一致。

## 5. Hero

Hero 高度从 `110svh`、`120svh`、`140svh` 中选择，表现为进入开放场域。必须包含：

* 姓名；
* 一条全局主介绍；
* 可选全局次级介绍；
* Room Number；
* 三至五个几何形体；
* 页面路径或继续阅读提示。

姓名和介绍可以位于不同空间点，例如姓名在右上、介绍在左下、Room Number 在中央空白，但核心信息必须在 Hero 范围内完整出现，并保留清楚的视觉联系。

形体可使用矩形、正方形、圆形、细线、十字和少量梯形。旋转角度只能从 `-28deg`、`-18deg`、`-8deg`、`0deg`、`12deg`、`22deg`、`32deg` 中选择；同屏最多出现两个主要倾斜角。

## 6. Navigation

导航可以拆分为空间中的独立文字节点：

```text
ENTRY     ARCHIVE     CONTACT     AGAIN     EN / 中文
```

节点不必位于同一行，但必须共享可追踪的轴线或形体关系，首屏必须有清楚入口。滚动后出现简化固定导航，保留当前阶段、语言切换和所有键盘功能。

hover/focus 只允许显示细线、轻度位置变化或字重变化。移动端恢复为清楚的纵向或两行列表，并通过非均匀间距保留空间感。

## 7. Personal Archive 与中段 section

中段不是连续卡片，而是一系列空间场。section 之间主要通过距离、锚点和少量跨边界几何体区分。跨 section 形体只能是装饰，不得承担核心文本或交互。

### 7.1 Projects

每个项目是独立空间对象，包含编号、名称、真实描述、技术和详情入口；字段可以位于不同空间点，但必须通过同一构图预设和语义关联保持可理解。

项目从以下模板选择：

* 漂浮三角关系：标题、描述和技术位于三个稳定空间点；
* 大形体关系：形体覆盖空白区域并接近内容，但不遮挡正文；
* 离心构图：标题靠近中心，其余信息向边缘延伸；
* 集中小岛：内容集中在小区域，周围保留大面积空白。

点击展开时，相邻形体让开，详情文本在预留空白区显现；不改变整个页面背景，不打开模态框，不让其他项目失去返回位置。

### 7.2 Experience

经历条目分布在不同高度，时间关系通过受控垂直距离和明确日期文本共同表达，不使用时间轴线。距离不得成为唯一时间说明。

### 7.3 Education

教育背景是不旋转的稳定锚点，具有清楚水平轴和紧凑内容，为其他漂浮模块提供阅读参照。

### 7.4 Working Stack

技术项可沿一条隐形圆弧、对角线或矩形边界排列。每个排列必须来自完整预设，不得逐项随机散落；视觉形态不代表熟练度或项目优先级。

### 7.5 Room Introduction / Exhibition Note

展厅说明是中段的独立空间场，由 seed 决定位置。它包含 Room Number、中英文名、历史时期、策展说明、当前空间构图名和 seed，只配一个小型几何体，并保留明显空白，形成阅读暂停点。

## 8. Contact

Contact 是固定末段的第一部分。Email、GitHub、LinkedIn 可以位于不同空间位置，但必须由共同轴线、细线或一致编号关系连接；链接使用完整文字且一眼可辨。

移动端恢复为清楚纵向排列。任何构图都不得让联系方式散落到多个视口、隐藏于 hover，或要求精确拖动才能访问。

## 9. Roll Again

Roll Again 固定在 Contact 后并作为页面最后一个主要 section。它可以表现为一个独立几何对象与明确文字标签的组合，而非通用按钮，但语义上必须使用真实 `button`。

辅助文案可从 `RELEASE THE FORMS`、`ANOTHER FIELD`、`NEW SPATIAL STATE`、`ROLL ANOTHER ROOM` 中选择，并同时保留清楚的全局功能说明。

激活后几何体沿其既有方向离开，文字淡出或移出，新构图从空白中建立；总时长不超过 `900ms`。Reduced Motion 下直接替换为新版本并返回 Hero。

## 10. 色彩关系

颜色只能通过全局令牌取得。Room 03 规定：

* 背景形成主要色场；
* 两至三个强调色可由几何体独立占据；
* 文字色保持稳定高对比；
* 同屏主要形体颜色不超过四种；
* 避免复杂透明叠加；
* 不用阴影模拟深度，深度来自尺度、位置和受限移动速度。

## 11. 动效、交互与可访问性

动效关键词为漂移、悬浮、离心、让位、释放和失重。只有少量形体可以进行低幅、长周期运动；正文、导航和交互标签必须静止。

* 视觉分散的项目字段必须通过 heading、`aria-labelledby` 或邻接 DOM 建立语义关系；
* 展开控件使用真实 `button` 与 `aria-expanded`；
* 装饰形体、重复文字和路径线使用 `aria-hidden="true"`；
* focus 不能依赖位置轻移，必须同时显示边线或下划线；
* 键盘顺序按 DOM 阅读顺序，不按屏幕坐标跳跃；
* 几何体不得捕获指针或遮挡交互目标。

### 11.1 Reduced Motion

Reduced Motion 下禁用持续漂移、视差、形体让位和离场动画；详情直接出现在预留空间，Roll Again 即时切换。最终静态构图必须仍有完整空间关系，任何内容都不能依赖运动才能被发现。

## 12. 响应式规则

移动端使用 4 列隐形网格和较少的纵向位置带，降低空白但不退化为卡片流：

* 核心内容回到清楚的纵向阅读次序；
* 每个空间场最多两个主要形体和一个辅助线；
* 旋转文字恢复水平，装饰形体仍可使用安全角度；
* Contact 链接纵向排列；
* 导航使用清楚列表并保留非均匀间距；
* section 高度允许降至 `80svh–120svh`；
* 不产生意外横向滚动。

中英文可分别调整位置预设、换行和留白比例，但核心信息、DOM 顺序与构图家族保持一致。

## 13. 确定性随机与上限

| 参数 | 安全集合或上限 |
| --- | --- |
| 整页空间预设 | 8 套完整构图 |
| section 高度 | `90svh` / `120svh` / `150svh` |
| Hero 高度 | `110svh` / `120svh` / `140svh` |
| 旋转角度 | `-28` / `-18` / `-8` / `0` / `12` / `22` / `32deg` |
| 同屏主要角度 | 最多 2 个 |
| Hero 形体 | 3–5 个；移动端 2–3 个 |
| 单个中段 section 形体 | 桌面端最多 3 个；移动端最多 2 个 |
| 持续移动形体 | 桌面端最多 2 个；移动端 0 个 |
| Roll Again 转场 | 不超过 `900ms` |

位置安全偏移只能来自预设的 `±1` 个网格单位，并通过碰撞和可读区域检查。所有选择必须来自 `deriveRng(seed, "room-03-layout")`、`room-03-decoration` 和 `room-03-motion` 等隔离随机流，禁止直接使用 `Math.random()`。

## 14. 验收标准

除[通用验收标准](../spec/26-acceptance-criteria.md)外，Room 03 还必须通过：

### 14.1 去色测试

* 去色后，尺度、距离、角度和空白仍形成明确视觉重量；
* 页面不依赖彩色形体才能区分标题、正文和链接；
* focus 与当前状态仍可辨。

### 14.2 静态测试

* 关闭全部动画后，形体和文字仍构成完整空间场；
* 正文、项目详情、Contact 与 Roll Again 不依赖漂移或让位才能访问；
* 页面不像随机散布或未加载完成。

### 14.3 风格与结构测试

* 大空白具有结构意义，正文仍容易找到；
* 不依赖容器边框，不像构成主义海报或生成式屏保；
* 任意合法 seed 都来自已验证预设，不发生内容碰撞或极端页面长度；
* Hero 固定在开头，Exhibition Note 只在中段变化，Contact → Roll Again 固定在结尾。
