# Room 02：构成主义 / Constructivism

## 1. 展厅身份

| 字段 | 值 |
| --- | --- |
| Room Number | `02` |
| Style ID | `constructivism` |
| 中文名 | 构成主义 |
| English Name | Constructivism |
| 历史时期 | 1910s–1930s |
| 一句话定义 | 一个由方向、命令、机械结构和强烈视觉张力推动的行动页面。 |

本展厅把同一份个人档案组织成一台传递信息、装配模块并推进阅读的机器。它的核心不是红黑配色，而是强方向、非对称张力、作为结构的文字和可见的装配逻辑。

本文件只定义 Room 02 的结构与表达；内容、配色和随机流仍由全局系统提供，并遵守[通用设计规范](../GENERATIVE_PORTFOLIO_DESIGN_SPEC.md)。

## 2. 核心原则

### 2.1 必须特征

* 整页存在一条清楚的主对角轴，但正文保持水平；
* 大编号、方向线、结构板和命令式辅助标签共同推动阅读；
* 非对称构图具有明确进入方向和停止点；
* section 像安装到主结构上的机械板，项目像设备、任务或系统模块；
* 动效表现装配、切入、锁定、撤离和执行；
* 去色后仍能通过方向、字号、边界与结构识别。

### 2.2 禁止特征

* 把红、黑或粗体当成风格的唯一依据；
* 旋转整页、旋转正文或让长段文字沿斜线阅读；
* 把所有文字都设为大写、粗体和倾斜；
* 圆角卡片、胶囊标签、柔软阴影、弹性动画或科幻扫描界面；
* 直接复制历史海报构图、口号或具体艺术作品；
* 让视觉方向覆盖真实内容或改变 DOM 阅读顺序。

## 3. 与易混淆展厅的区别

### 3.1 构成主义 ≠ 至上主义

构成主义用方向轴、机械边界和命令式层级推进阅读；至上主义依靠开放距离、漂浮形体与失重关系。Room 02 的几何体安装在信息结构上，不是自由悬浮物。

### 3.2 构成主义 ≠ 风格派

构成主义允许受控对角线、切角和局部倾斜；风格派只允许水平与垂直的正交组织。Room 02 的运动沿主对角轴切入和撤离，而不是重新分配正交网格。

### 3.3 构成主义 ≠ 通用粗野主义

粗线、大字和高对比本身不足以成立。每条方向线必须连接信息或划分结构，每个动作词只能作为辅助视觉文本，不能靠未经整理的冲突制造“粗粝感”。

## 4. 整页结构

页面遵守“首尾固定、中间可变”：

```text
Hero（固定）
↓
Projects / Experience / Education / Stack / Exhibition Note
（由 seed 决定顺序）
↓
Contact（固定）
↓
Roll Again（固定且位于最后）
```

桌面端主轴角度只能从 `-12deg`、`-8deg`、`8deg`、`12deg` 中选择。只有装饰结构、标题容器、辅助线和局部项目边界可以沿主轴倾斜；正文、链接正文和表格式事实必须水平。

中段 section 从以下模板中选择：

* 斜向分割：方向边界划分标题区和内容区；
* 结构重叠：标题板压住内容边缘，但不遮挡正文；
* 垂直命令带：大编号与动作词在一侧，正文在另一侧。

同一模板不得连续出现三次。中段顺序、模板和方向都必须来自 Room 02 的独立 seeded 随机流，DOM 顺序必须与视觉阅读顺序一致。

## 5. Hero

Hero 高度为 `100svh`，视觉中心是“姓名 + 方向线 + 圆形结构 + 命令辅助词”的组合。必须包含：

* 姓名和全局 Hero 文案；
* 个人定位与可选全局次级文案；
* 一至两条对角结构线；
* 一个圆形或半圆结构；
* Room Number；
* 页面滚动方向提示；
* 一个不改变个人事实的辅助行动词。

辅助词只能从 `BUILD`、`STRUCTURE`、`EXECUTE`、`ASSEMBLE`、`SYSTEM`、`MODULE` 中 seeded 选择，并标记为装饰文本，不能替代或改写真实介绍。

姓名可以横跨页面、穿过结构线、分为两行或在视觉上局部超出视口，但必须保留完整可见和可访问版本。Hero 只能有一个最主要大标题；其他层级通过编号、字号、方向和大小写建立，不能全部同强度竞争。

## 6. Navigation

导航是一条直角“命令带”：

```text
[01 ENTRY] [02 ARCHIVE] [03 CONTACT] [04 AGAIN] [EN / 中文]
```

当前阶段使用填充状态与 `aria-current`；hover/focus 时背景块沿主轴移动。移动端可横向滚动，但必须显示滚动提示、支持键盘，并保证每个目标至少具有足够触摸尺寸。不得把汉堡菜单作为唯一入口。

## 7. Personal Archive 与中段 section

每个 section 入口包含巨大编号、中英文名称、一句命令式辅助标签和一个对角切口。辅助标签只描述界面动作，不新增个人事实。

### 7.1 Projects

每个项目以设备或系统模块呈现，包含编号、名称、类型、角色、真实描述、技术和可选状态字段。状态只能来自内容数据，不得虚构完成度。

项目从以下安全模板选择：

* 标题左上、描述右下；
* 标题纵向视觉排列、描述保持水平；
* 编号占据约三分之一结构宽度；
* 一个大圆或半圆穿过项目边缘；
* 项目标题横跨两个结构区。

边界可使用切角矩形、对角分割、半圆结构、粗线和编号条，禁止圆角卡片。连续项目不得全部朝同一方向倾斜。

展开时项目板沿主轴滑出，详情从下方或侧面进入，其他项目保持文档流位置；再次激活时机械式收回。禁止模态框，正文展开后仍保持水平。

### 7.2 Experience

经历表现为任务记录，以编号、角色、组织、时期和职责组成结构板；不用传统时间轴。

### 7.3 Education

教育背景表现为“训练与基础结构”模块，必须完整呈现学校、专业、学位和预计完成时间，不使用军衔、权限或其他虚构标签。

### 7.4 Working Stack

技术项可置于不同长度的机械条中，但条长只用于构图，不能代表熟练度、使用频率或优先级。每项仍需以真实文字可读。

### 7.5 Room Introduction / Exhibition Note

展厅说明是中段的独立 section，由 seed 决定位置。它以简短宣言开场，再显示完整策展说明、Room Number、中英文名、历史时期、布局变体和 seed。宣言只描述结构原则，不能替代真实说明或个人介绍。

## 8. Contact

Contact 是固定末段的第一部分，设计成清楚的“传输终端”。可以使用 `TRANSMIT`、`OPEN A CHANNEL` 或 `CONTACT / CONNECT` 作为辅助标题，但 Email、GitHub、LinkedIn 和必要职业链接必须以真实文字呈现。

不得加入雷达、扫描、假终端、输入命令或科幻 UI。链接必须能直接点击、复制并通过键盘访问。

## 9. Roll Again

Roll Again 固定在 Contact 后并作为页面最后一个主要 section。它占据接近整屏宽度的横向结构板，包含大号动作文字、对角切口、当前 seed 和明确的“生成另一个版本”说明。

辅助动作词可从 `REASSEMBLE`、`NEW CONSTRUCTION`、`REBUILD THE PAGE`、`EXECUTE ANOTHER VERSION` 中选择。

激活后模块沿当前主轴撤离、结构线收缩，新模块从相反方向进入；总转场为 `500–900ms`。必须先更新 URL 状态并支持浏览器前进后退；Reduced Motion 下不播放撤离序列。

## 10. 色彩关系

颜色只能通过全局令牌取得。Room 02 规定：

* 背景与结构色保持高对比；
* 一个主要强调色占据一块大结构面；
* 第二强调色只用于圆形、编号或命令带；
* 颜色不得掩盖文字、focus 或当前状态；
* 不写死红黑，不使用渐变；
* 低张力的相邻色只能用于非关键小面积，不得削弱结构边界。

## 11. 动效、交互与可访问性

动效关键词为装配、切入、推进、锁定、撤离和执行。曲线应快速开始、明确停止，以少量线性或接近线性的运动为主，避免柔软缓动和弹性回弹。

* 正文和链接始终水平；
* 展开控件使用真实 `button` 与 `aria-expanded`；
* 装饰行动词、重复标题与几何体使用 `aria-hidden="true"`；
* 被形体视觉穿过的姓名保留一个完整语义文本；
* focus 同时使用填充、边界或位移反馈，不能只依赖颜色；
* 横向命令带提供键盘滚动和当前项状态；
* 所有移动方向都不得改变 DOM 语义顺序。

### 11.1 Reduced Motion

Reduced Motion 下禁用沿轴撤离、背景块滑动和结构线收缩；项目详情直接显现，Roll Again 使用即时状态替换。方向、切口、编号和层级必须在静态状态中完整保留。

## 12. 响应式规则

移动端将主轴视觉角度限制在 `-6deg`、`-3deg`、`3deg`、`6deg`，正文全部恢复水平，并保留：

* 对角分区；
* 强编号和单一主要标题；
* 方向箭头与机械切换感；
* 可横向访问的命令带；
* 足够的触摸目标和明确 focus；
* 不遮挡正文的缩小圆形与结构线。

移动端每屏主要对角结构最多两条，不出现意外横向溢出。中英文可分别调整字号、换行和结构板高度，但不得改变事实、section 顺序或交互职责。

## 13. 确定性随机与上限

| 参数 | 安全集合或上限 |
| --- | --- |
| 桌面主轴 | `-12deg` / `-8deg` / `8deg` / `12deg` |
| 移动主轴 | `-6deg` / `-3deg` / `3deg` / `6deg` |
| section 模板 | 3 种；不得连续 3 次相同 |
| 项目模板 | 5 种；连续方向必须受平衡规则约束 |
| Hero 结构线 | 1–2 条 |
| 单屏主要形体 | 最多 3 个 |
| 单个 section 装饰结构 | 最多 4 个 |
| 持续动画 | `0` 个 |
| Roll Again 转场 | `500–900ms` |

所有选择必须来自 `deriveRng(seed, "room-02-layout")`、`room-02-decoration` 和 `room-02-motion` 等隔离随机流，禁止直接使用 `Math.random()`。

## 14. 验收标准

除[通用验收标准](../spec/26-acceptance-criteria.md)外，Room 02 还必须通过：

### 14.1 去色测试

* 去色后，主轴、编号、切口、结构板和阅读方向仍清楚；
* 不依赖红黑即可识别；
* 当前导航、链接和 focus 状态仍可辨。

### 14.2 静态测试

* 关闭动画后，每个模块仍像已装配完成的结构，而不是失去定位；
* 项目详情、导航与 Roll Again 无动画也能完整使用；
* 正文不依赖进入动画才能出现。

### 14.3 风格与结构测试

* 对角线承担分区或方向职责，正文保持水平可读；
* 页面有机械行动感，但不是海报复制、粗野换肤或科幻界面；
* 与至上主义和风格派有明确结构差异；
* Hero 固定在开头，Exhibition Note 只在中段变化，Contact → Roll Again 固定在结尾。
