# Room 01：纯粹主义 / Purism

## 1. 展厅身份

| 字段 | 值 |
| --- | --- |
| Room Number | `01` |
| Style ID | `purism` |
| 中文名 | 纯粹主义 |
| English Name | Purism |
| 历史时期 | 约 1918–1925 |
| 一句话定义 | 一个由稳定比例、低信息密度和少量精确几何体组成的安静机器空间。 |

本展厅把同一份个人档案放置在精密、克制且可复现的数字平面上。它不追求泛化的“极简高级感”，而强调所有元素都经过整理、所有比例都有明确职责、几何形体与内容共同形成数字静物。

本文件只定义 Room 01 的结构与表达；内容、配色和随机流仍由全局系统提供，并遵守[通用设计规范](../GENERATIVE_PORTFOLIO_DESIGN_SPEC.md)。

## 2. 核心原则

### 2.1 必须特征

* 一条垂直信息轴与一条水平比例轴；每个主要模块至少对齐其中一条；
* 非完全居中的稳定主内容平面，桌面端宽度为视口的 `76%–88%`，允许向一侧偏移 `4%–8%`；
* 低信息密度、大面积安静平面和少量承担结构职责的几何体；
* “标签面 + 内容面”的 section 关系；
* 直角、细线、精确比例和无弹性的校准式交互；
* 去色和关闭动画后，仍可通过比例、轴线与留白识别。

### 2.2 禁止特征

* 把所有内容放进圆角卡片；
* 悬浮胶囊导航、玻璃模糊、渐变或透明叠色；
* 无职责的几何装饰、随机旋转、漂浮、弹跳或抖动；
* 依赖奢侈品式大留白、衬线字与低对比来假装“高级”；
* 用技术标签云、技能 Logo 墙或时间轴圆点组织档案；
* 让同一页面出现三种以上 section 构图关系。

## 3. 与易混淆展厅的区别

### 3.1 纯粹主义 ≠ 至上主义

纯粹主义的几何体锁定在稳定轴线和比例关系上，不制造失重与离心感；至上主义依赖开放距离、漂浮关系与受控倾斜。Room 01 禁止随机旋转，且主要内容始终有可追踪的对齐基准。

### 3.2 纯粹主义 ≠ 风格派

纯粹主义可以使用隐含轴线和空白平面，不要求网格线持续可见；风格派必须用正交线真正切割并分配空间。Room 01 的重点是少量对象的比例平衡，而不是矩形网格建筑。

### 3.3 纯粹主义 ≠ 通用极简主义

删减装饰或放大留白不足以构成本展厅。几何体必须承担对齐、配重、延伸或版本标记等职责，项目也必须使用专属平面关系，不能退化为普通极简作品集。

## 4. 整页结构

页面遵守“首尾固定、中间可变”：

```text
Hero（固定）
↓
Projects / Experience / Education / Stack / Exhibition Note
（由 seed 决定顺序）
↓
Contact（固定）
↓
Roll Again（固定且位于最后）
```

中段 section 使用以下三种关系：

1. 标签左、内容右；
2. 内容左、标签右；
3. 标签在上、内容在下。

同一版本最多选择两种关系。标签面占 `18%–28%`，内容面占 `60%–75%`，二者之间优先保留空白平面，而不是为每处添加分割线。

中段顺序、section 关系和偏移都必须来自 Room 01 的独立 `layout` 随机流；视觉顺序必须与 DOM 顺序一致。

## 5. Hero

Hero 高度只能从 `90svh`、`100svh`、`120svh` 中选择，并包含：

* 姓名；
* 一条全局主介绍；
* 可选的一条全局次级介绍；
* 低调的 Room Number 或 seed 标记；
* 两至四个几何体；
* 向下阅读提示。

姓名不完全居中，位于左上至中部或其镜像位置，占页面宽度的 `40%–60%`。建议字号为：

```css
font-size: clamp(3.5rem, 8vw, 9rem);
line-height: 0.95;
font-weight: 500;
```

可在 `500–700` 范围选择字重；介绍文字控制在 `28–42ch`。

几何体职责固定为：

* 大矩形：控制 Hero 主要比例；
* 圆形：提供次要视觉重量；
* 细长矩形：连接文字与页面边缘；
* 小方块：标记版本、seed 或 Room Number。

Hero 只允许以下 seeded 变体：文字平面左/右、圆形上/下、长矩形横/竖、三种高度、姓名一/两行。禁止随机旋转；同屏主要几何体不得超过四个。

## 6. Navigation

导航是一条低高度铭牌，不使用悬浮胶囊：

```text
STARK LIN       ROOM 01       ENTRY / ARCHIVE / CONTACT / AGAIN       EN / 中文
```

导航可固定在顶部或嵌入首屏；无背景模糊，使用细线和间距分隔。滚动后可收缩，但必须保留当前阶段、语言切换和键盘可达入口。

移动端采用两至三行铭牌，显示姓名、`01`、阶段入口和语言切换；菜单即使折叠也不得把汉堡图标作为唯一说明。

## 7. Personal Archive 与中段 section

所有中段 section 都延续标签面、内容面和安静平面的关系，但空白比例可随模板变化。section 编号、名称、可选年份与分类放在标签面；正文保持普通阅读方向和合理行宽。

### 7.1 Projects

项目不是卡片，而是一组精确平面关系。每个项目包含编号、名称、年份、真实描述、角色和技术，并从以下模板中选择：

* 水平静物：名称在左、描述在中、年份与技术在右，下方放置比例矩形；
* 上下平衡：大标题在上、两列描述在下，几何体位于标题末端；
* 单体平面：内容集中在稳定区域，旁边保留大面积空白。

同一页面最多混合两种项目模板。项目大小不得暗示虚构优先级。

点击展开时，当前项目沿文档流纵向扩展，其他项目自然下移，相关几何体同步调整比例。禁止模态框；过渡为 `300–500ms` 的直线位移和尺寸变化。

### 7.2 Experience

经历使用连续横向平面；年份、角色、组织与说明分别对齐到稳定轴线，不使用时间轴圆点或模板式竖线。

### 7.3 Education

教育背景是一份紧凑、平静的档案条目，可用一条长矩形建立时间方向，但不得弱化学校、学位和日期事实。

### 7.4 Working Stack

技术栈使用精确索引或水平清单，例如 `LANGUAGE / Go`、`DATABASE / SQL`。不得使用标签云、胶囊或用条形长度表达不存在的熟练度。

### 7.5 Room Introduction / Exhibition Note

展厅说明是中段的独立 section，由 seed 决定位置。其外观接近博物馆作品标签，但不模拟纸张或贴纸，必须包含 Room Number、中英文名、历史时期、60–140 字中文或 40–90 词英文策展说明、当前构图变体和 seed。视觉层级低于项目标题。

## 8. Contact

Contact 是固定末段的第一部分，占 `80svh–120svh` 的开放平面。Email、GitHub、LinkedIn 与必要职业链接使用完整文字，保持高可读性和清晰 focus 状态，不使用社交图标。

该 section 应比 Archive 更安静；联系方式必须一眼可见，不得要求展开、拖动或解谜。

## 9. Roll Again

Roll Again 固定在 Contact 后并作为页面最后一个主要 section。它被表现为“重新校准机器”的控制器：长条直角矩形、明确边框、动作文字位于一端、当前 seed 位于另一端。

主文案可以从展厅辅助标签池选择 `RECOMPOSE`、`CALIBRATE AGAIN` 或 `NEXT CONFIGURATION`，但必须同时显示清楚的全局功能说明。

hover/focus 时内部比例块沿单一轴线移动；激活后当前几何体先对齐或归零，再加载新版本。转场不得阻塞导航，Reduced Motion 下立即切换最终状态。

## 10. 色彩关系

颜色只能通过全局令牌取得。Room 01 规定关系而不写死色值：

* 背景色占页面 `65%–85%`；
* 主要文字使用稳定的单一文本色；
* 主要强调色只用于一至两个结构几何体；
* 第二强调色只占小面积；
* 同屏主要颜色不超过四种；
* 不使用渐变、阴影造深度或透明叠色。

## 11. 动效、交互与可访问性

动效关键词为校准、滑入、对齐、比例建立和平面扩展。即时反馈 `100–180ms`，普通过渡 `180–400ms`，结构展开不超过 `500ms`。

* 所有展开控件使用真实 `button`，并同步 `aria-expanded`；
* 视觉拆分的姓名和文案保留单一、完整的可访问文本；
* 装饰几何体使用 `aria-hidden="true"`；
* hover 行为必须有 focus 与触摸等价反馈；
* focus 样式必须同时使用边框、线条或位置变化，不能只改颜色；
* 页面不得因绝对定位改变屏幕阅读器顺序。

### 11.1 Reduced Motion

Reduced Motion 下禁用滑入和比例块移动；展开直接到达最终尺寸，Roll Again 不播放归零序列。静态构图、内容和 focus 反馈必须全部保留。

## 12. 响应式规则

移动端不能退化成普通单列卡片：

* 保留一条明确的竖向比例轴；
* 几何体缩小并位于文字之间，而不是全部退为背景；
* section 标签占独立窄条或上方比例面；
* 项目描述回到自然文档流；
* 页面边距为 `16–24px`，核心文字不得裁切；
* 主要几何体上限从四个降为三个；
* 不产生意外横向滚动。

中英文可以独立调整换行、字距和标签面宽度，但内容、DOM 顺序和构图家族必须一致。

## 13. 确定性随机与上限

Room 01 只能从经过设计的有限集合中选择：

| 参数 | 安全集合或上限 |
| --- | --- |
| Hero 高度 | `90svh` / `100svh` / `120svh` |
| 主平面偏移 | 左或右 `4%` / `6%` / `8%` |
| section 关系 | 三种模板，每页最多两种 |
| 项目模板 | 三种模板，每页最多两种 |
| Hero 主要几何体 | 桌面端 2–4；移动端 2–3 |
| 单个 Archive section 装饰 | 最多 2 个 |
| 旋转角度 | 仅 `0deg` |
| 持续动画 | `0` 个 |

所有选择必须来自 `deriveRng(seed, "room-01-layout")`、`room-01-decoration` 和 `room-01-motion` 等隔离随机流。禁止直接调用 `Math.random()`。

## 14. 验收标准

除[通用验收标准](../spec/26-acceptance-criteria.md)外，Room 01 还必须通过：

### 14.1 去色测试

* 移除全部颜色后，双轴、比例面、配重关系和留白仍然清楚；
* 页面不能因失去强调色而变成普通单列作品集；
* 链接和 focus 仍通过线条、位置或字重可辨。

### 14.2 静态测试

* 关闭全部动画后，Hero 与每个 section 都保持完整构图；
* 项目详情无需动画也能展开和收起；
* Roll Again 的功能说明和当前状态仍然清楚。

### 14.3 风格与结构测试

* 页面安静但不空洞，几何体承担真实结构职责；
* 不像奢侈品牌官网，也不与至上主义或风格派混淆；
* 任意合法 seed 都只产生同一家族内的构图；
* Hero 固定在开头，Exhibition Note 只在中段变化，Contact → Roll Again 固定在结尾。
